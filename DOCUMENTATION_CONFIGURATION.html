✨ 6. Nouvelles Fonctionnalités Ajoutées

📱 Mobile Money avec sélection d'opérateur

Quand l'agent sélectionne "Mobile Money" comme mode de paiement, 4 boutons d'opérateur apparaissent :

OpérateurCouleurStocké en DB
🟠 Orange Money#FF6600payment_operator = 'orange'
🔵 Moov Money#0057A8payment_operator = 'moov'
🟦 Wave#1BC5BDpayment_operator = 'wave'
🟡 MTN MoMo#FFCC00payment_operator = 'mtn'

Disponible dans : Piscine (tickets), Restaurant (commandes), Hôtel (réservations)

📷 Scanner QR code pour paiement

Après avoir sélectionné un opérateur Mobile Money, la caméra du téléphone s'ouvre automatiquement pour scanner le QR code du client.

  • Utilise la bibliothèque jsQR (lecture en temps réel)
  • Fonctionne avec la caméra arrière du téléphone
  • La référence de paiement scannée est enregistrée dans le champ payment_reference
  • Affiche une confirmation visuelle avec la référence tronquée

📸 Photo de scène pour incidents (Piscine)

Lors du signalement d'un incident, l'agent peut prendre une photo directement depuis son téléphone.

  • La photo est compressée automatiquement (max 1024×768px, qualité 75%)
  • Stockée en base de données sous forme base64 dans le champ photo_url
  • Prévisualisation avec zoom et option de suppression
  • Champ optionnel — ne bloque pas la saisie si pas de photo

🔧 7. Migrations Base de Données

Toutes ces modifications ont été appliquées directement sur Supabase :

Colonnes ajoutées à la table tickets

ALTER TABLE tickets
  ADD COLUMN IF NOT EXISTS payment_operator VARCHAR(20),
  ADD COLUMN IF NOT EXISTS payment_reference VARCHAR(200);

Colonnes ajoutées à la table sales (restaurant)

ALTER TABLE sales
  ADD COLUMN IF NOT EXISTS payment_operator VARCHAR(20),
  ADD COLUMN IF NOT EXISTS payment_reference VARCHAR(200);

Colonnes ajoutées à la table reservations (hôtel)

ALTER TABLE reservations
  ADD COLUMN IF NOT EXISTS payment_operator VARCHAR(20),
  ADD COLUMN IF NOT EXISTS payment_reference VARCHAR(200),
  ADD COLUMN IF NOT EXISTS cni_number VARCHAR(50),
  ADD COLUMN IF NOT EXISTS origin_city VARCHAR(100),
  ADD COLUMN IF NOT EXISTS destination_city VARCHAR(100);

Colonnes ajoutées à la table incidents

ALTER TABLE incidents
  ADD COLUMN IF NOT EXISTS photo_url TEXT;

Colonnes ajoutées à la table events

ALTER TABLE events
  ADD COLUMN IF NOT EXISTS price DECIMAL(12,2) DEFAULT 0,
  ADD COLUMN IF NOT EXISTS deposit_paid DECIMAL(12,2) DEFAULT 0;

Corrections sur d'autres tables

-- payrolls : colonne manquante
ALTER TABLE payrolls ADD COLUMN IF NOT EXISTS paid_by INTEGER;

-- expenses : colonnes manquantes
ALTER TABLE expenses
  ADD COLUMN IF NOT EXISTS reference VARCHAR(200),
  ADD COLUMN IF NOT EXISTS payroll_id INTEGER;

-- user_layouts : colonne manquante
ALTER TABLE user_layouts
  ADD COLUMN IF NOT EXISTS layout_type VARCHAR(50) NOT NULL DEFAULT 'reports';

🐛 8. Corrections Effectuées

Correction critique

Tableau de bord ne se chargeait pas

La table events n'avait pas les colonnes price et deposit_paid. Sequelize générait un SELECT avec toutes les colonnes du modèle, causant une erreur PostgreSQL "column does not exist" → erreur 500 → le tableau de bord affichait "Erreur lors du chargement". Résolu en ajoutant les colonnes manquantes.

Correction modèles

ENUM → STRING dans tous les modèles Sequelize

17 fichiers de modèles utilisaient DataTypes.ENUM incompatible avec PostgreSQL sur Supabase. Tous remplacés par DataTypes.STRING. Cela corrigeait des erreurs de démarrage du backend.

Correction mobile

Header trop chargé sur téléphone

Sur mobile, la barre du haut affichait simultanément : hamburger, titre, badge sync, chip "En ligne", nom complet, rôle, avatar — causant un débordement. Résolu : nom/rôle et chip masqués sur petit écran, seule l'icône wifi s'affiche.

Correction build

Erreurs TypeScript bloquant le déploiement

Les nouveaux composants PaymentSelector.tsx et QRScanner.tsx avaient des imports inutilisés (Dialog, Chip, FlipIcon…). TypeScript les traitait comme erreurs TS6133, faisant échouer le build sur Render. Résolu en supprimant les imports non utilisés.

Correction tables manquantes

Colonnes absentes dans payrolls, expenses, incidents

Plusieurs tables n'avaient pas toutes les colonnes définies dans les modèles Sequelize. Un script de migration a été exécuté pour aligner la DB avec les modèles.
