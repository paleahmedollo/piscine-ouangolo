Qu'est-ce que l'UAT ?

L'UAT (User Acceptance Testing) est un environnement identique à la production où tu peux tester toutes les nouvelles fonctionnalités sans risquer d'affecter les données réelles. Quand les tests sont validés, on fusionne vers la production.

Partie 4 — Workflow quotidien
6
Comment travailler au quotidien
Développer sur staging → Tester → Merger en production

🧪 Étape 1 — Développer et tester sur staging

  1. Basculer sur la branche staging : git checkout staging
  2. Faire les modifications dans le code
  3. Pousser : git add . && git commit -m "..." && git push origin staging
  4. Render UAT se redéploie automatiquement (~3 min)
  5. Tester sur piscine-ouangolo-frontend-uat.onrender.com

✅ Étape 2 — Valider et envoyer en production

  1. Si les tests sont OK, basculer sur main : git checkout main
  2. Fusionner staging dans main : git merge staging
  3. Pousser en production : git push origin main
  4. Render PROD se redéploie automatiquement (~3 min)
  5. Vérifier sur la prod que tout fonctionne ✅
⚠️ Important : Les données de l'UAT sont indépendantes de la production. Tu peux créer/supprimer des données de test librement sans aucun impact sur les vrais clients.
